# Деплой Telegram-бота «Щегол»

## Быстрый вариант для показа: Railway

Railway подходит, если нужно быстро показать бота завтра и не покупать VPS.

1. Загрузите проект `telegram-bot` в GitHub.
2. Откройте Railway: https://railway.com
3. Создайте новый проект: `Deploy from GitHub repo`.
4. Выберите репозиторий с ботом.
5. В `Variables` добавьте переменную:

```env
BOT_TOKEN=токен_бота_из_BotFather
```

6. Railway сам увидит `Dockerfile` и запустит бота.
7. В логах после деплоя должно быть примерно:

```text
Bot started: @Shchegol_coffee_bot
Start polling
```

Важно: не запускайте этот же бот одновременно на компьютере. Telegram polling должен работать только в одном месте.

## Что уже подготовлено

- `Dockerfile` - запуск бота в контейнере.
- `railway.json` - настройки деплоя Railway.
- `.dockerignore` - исключает `.env`, `.venv`, логи, SQLite и локальные фотографии.
- `requirements.txt` - зависимости проекта.

## Варианты хостинга

### Railway

Лучший вариант для быстрого демо.

Плюсы:
- быстрый деплой из GitHub;
- переменные окружения для токена;
- Docker работает из коробки;
- можно быстро показать проект заказчику.

Минусы:
- бесплатность ограничена trial/кредитами;
- это не вечный бесплатный сервер.

### Koyeb

Подходит как бесплатная альтернатива для небольшого сервиса.

Плюсы:
- есть free tier;
- можно запускать Docker/GitHub-проект.

Минусы:
- на бесплатных тарифах возможны ограничения и засыпание сервиса;
- для демонстрации нормально, для продажи лучше потом выбрать стабильный тариф.

### Render

Можно использовать, но для Telegram polling я бы не выбирал первым.

Плюсы:
- простой деплой;
- понятный интерфейс.

Минусы:
- бесплатные сервисы могут засыпать;
- при неправильной настройке легко получить конфликт, если бот запущен в двух местах.

### PythonAnywhere

Удобен для простых Python-проектов, но не лучший вариант для постоянно работающего polling-бота на бесплатном тарифе.

## Важный момент про базу

Сейчас корзина, заказы и карта гостя хранятся в SQLite. Для демо этого достаточно.

Но на бесплатном облачном хостинге данные могут сбрасываться при пересборке или перезапуске контейнера, если не подключить постоянный диск/volume.

Для настоящего запуска лучше потом сделать один из вариантов:

- Railway Volume + SQLite;
- PostgreSQL на Railway/Supabase;
- VPS + SQLite/PostgreSQL.

## Почему локальные фотографии не отправляем

В Docker не загружаются локальные фотографии из папки `assets`. Бот использует публичные URL в `data.py`, поэтому изображения будут видны другим людям в Telegram.
